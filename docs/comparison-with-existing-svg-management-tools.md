# SVG管理ツールの比較検討資料

本ドキュメントでは、SVGアセットを管理する既存の主要ツールと、個人開発に特化して設計された「SVG Manager」の機能・アーキテクチャの客観的な比較を行います。

Gemini 3.1 proによって作成されたドキュメントです / Written by Gemini 3.1 pro

## 1. 概要とターゲット層の違い

SVGやアイコンを管理するツールは多数存在しますが、設計思想によって最適なユースケースが異なります。

* **既存の主要ツール (Eagle, Nucleo, IconJarなど):**
  主に「デザイナー」をターゲットとしており、ローカルストレージ上の大量の画像ファイル（.svg, .pngなど）を視覚的に整理し、ドラッグ＆ドロップでデザインツールへ出力することに特化しています。
* **SVG Manager:**
  主に「開発者」をターゲットとしています。LLM（大規模言語モデル）の出力結果など、テキストベースで生成されたSVGコードを直接ブラウザに貼り付けて保存・プレビューするフローに特化しています。

## 2. 機能およびアーキテクチャの比較表

各ツールの特性を以下の表にまとめます。

| 比較項目 | SVG Manager | Eagle | Nucleo | IconJar |
| :--- | :--- | :--- | :--- | :--- |
| **提供形態** | Webアプリケーション (SPA) | デスクトップアプリ | デスクトップ / Webアプリ | デスクトップアプリ |
| **主要な入力方法** | SVGコードの直接貼り付け | ファイルのインポート | ファイルのインポート | ファイルのインポート |
| **データ管理場所** | クラウド (Firestore) | ローカル | ローカル / クラウド | ローカル |
| **クラウド同期** | 標準搭載 (リアルタイム自動同期) | 外部ストレージ依存 (Google Drive等) | 標準搭載 (Premium版) | 外部ストレージ依存 |
| **OS依存** | なし (ブラウザベース) | Windows / macOS | Windows / macOS / Web | macOS専用 |
| **アクセス制御** | 指定UIDのみに限定する厳格な制限 | ローカルPCのセキュリティに依存 | アカウントベース (チーム共有可) | ローカルPCのセキュリティに依存 |

## 3. 各ツールの詳細な特性

### 3.1. 既存ツール (デザイナー・ファイル管理向け)

**Eagle / IconJar**
これらはローカルPCに保存されたファイルをタグ付けやフォルダ階層で管理する能力に優れています。しかし、クラウドで複数端末間の同期を行う場合、別途サードパーティのクラウドストレージ（Dropbox等）を設定し、データベースファイルを同期させる必要があります。また、IconJarのように特定のOS（macOS）に限定されるケースもあります。

**Nucleo**
自社のアイコンライブラリとユーザーのカスタムアイコンを統合管理できる強力なツールです。Webアプリ版も存在しクラウド同期も可能ですが、多機能であるがゆえに、単にコードスニペットを保存したい開発者にとってはオーバースペックとなる場合があります。

### 3.2. SVG Manager (開発者・コード管理向け)

**ファイルシステムを汚染しないコードベースの管理**
既存ツールが「.svgファイル」を管理するのに対し、SVG Managerは「SVGコード（文字列）」をデータとして管理します。LLMが生成したSVGコードをそのまま貼り付けるだけで即座にプレビューされるため、ローカルに一時ファイルを作成する手間とファイル散乱のリスクを排除します。

**環境非依存の完全なクラウドネイティブ**
Firestoreを用いたリアルタイム同期を標準で備えており、Googleアカウントによる認証を経ることで、端末やOSを問わずどこからでもアセットにアクセス可能です。

**個人利用に特化したセキュリティ設計**
パブリックなSaaSとは異なり、個人専用ツールとしての安全性を担保しています。Identity Platformによる新規ユーザー作成の無効化 や、デプロイ時に環境変数から管理者UIDを注入してFirestoreのセキュリティルールを動的生成する仕組み により、第三者のアクセスを完全に遮断しています。

## 4. 結論

* **既存ツールが適しているケース:** 大量のデザインファイル（SVG, PNG, JPG等）をローカル環境で視覚的に整理し、デザインソフトウェア（FigmaやIllustratorなど）と連携させたい場合。
* **SVG Managerが適しているケース:** 開発プロセスのなかで、LLMが生成したSVGやWeb上のSVGコードスニペットを直接保存・管理し、OSに依存しないセキュアなクラウド環境で一元化したい場合。

SVG Managerは、開発者の「コードから直接アセットを管理する」という要件に対し、無駄を省いたセキュアな解決策を提供します。
